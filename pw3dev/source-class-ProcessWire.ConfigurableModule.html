  4:  * ProcessWire ConfigurableModule Interface
  5:  *
  6:  * Provides the base interfaces required by modules.
  7:  * 
  8:  * This file is licensed under the MIT license
  9:  * https://processwire.com/about/license/mit/
 10:  *
 11:  * ProcessWire 3.x, Copyright 2016 by Ryan Cramer
 12:  * https://processwire.com
 13:  *
 14:  * 
 15:  * About the ConfigurableModule interface
 16:  * ======================================
 17:  * ConfigurableModule is an interface that indicates the module is configurable by providing
 18:  * __get() and __set() methods for getting and setting config values. Modules implementing 
 19:  * this interface are assumed to also implement the 'Module' interface.
 20:  * 
 21:  * The module must also provide one (1) of the following:
 22:  *
 23:  * 1. A getModuleConfigInputfields([$data]) method (static or non-static); OR
 24:  * 2. A separate ModuleName.config.php file that just populates $config array; OR
 25:  * 3. A separate ModuleNameConfig.php file that contains a ModuleConfig class.
 26:  *
 27:  * For more details about the above options, see the commented methods within
 28:  * the interface.
 29:  *
 30:  * When you use this as an interface, you MUST also use 'Module' as an interface,
 31:  * i.e. "class Something implements Module, ConfigurableModule"
 32:  *
 33:  * Hint: Make your ConfigurableModule classes inherit from WireData, which already has
 34:  * the get/set required methods.
 35:  *
 36:  * You may optionally specify a handler method for configuration data: setConfigData().
 37:  * If present, it will be used. See commented function reference in the interface below.
 38:  * 
 39:  *
 40:  */
 41: interface ConfigurableModule {
 42: 
 43:     /**********************************************************************************
 44:      * getModuleConfigInputfields method (static or non-static)
 45:      *
 46:      * Return an InputfieldWrapper of Inputfields used to configure the class. This may
 47:      * be specified either as a static method or a non-static method.
 48:      *
 49:      * Benefits of static version
 50:      * ===========================
 51:      * 1. The module does not need to be instantiated in order to configure it, which
 52:      *    means that unnecessary hooks won't get attached and unnecessary assets won't
 53:      *    be triggered to load.
 54:      * 2. It is supported by all versions of ProcessWire.
 55:      *
 56:      * Drawbacks of static version
 57:      * ===========================
 58:      * 1. You cannot pull config values directly from the module since it isn't
 59:      *    instantiated, and thus you must use the provided $data array. This $data array
 60:      *    only contains values if the module has been configured before.
 61:      * 2. You can't access $this or anything you'd typically pull from it, like API vars
 62:      *    or translation methods.
 63:      *
 64:      * Benefits of non-static version
 65:      * ==============================
 66:      * 1. You are working with the module in the same context that it is when running,
 67:      *    thus you can pull config values and API vars directly from $this->something.
 68:      * 2. It can be extended in descending classes.
 69:      * 3. The $data argument can be omitted, as you don't need it since all config
 70:      *    properties can be accessed directly from $this->[any property]. 
 71:      * 4. You can specify an optional $inputfields argument in your function definition
 72:      *    and if present, ProcessWire will prepare an InputfieldWrapper for you, saving
 73:      *    a step. When present, you can optionally omit the return statement at the 
 74:      *    bottom of the method as well. 
 75:      *
 76:      * Drawbacks of non-static version
 77:      * ================================
 78:      * 1. It is supported only in ProcessWire versions 2.5.27 or newer.
 79:      * 2. The module must be instantiated in order to configure it, so it may trigger
 80:      *    load of any used assets or attachment of any hooks unnecessarily.
 81:      *
 82:      * @param array $data Array of config values indexed by field name (static version only)
 83:      *  Note that this array will be empty if the module has not been configured before.
 84:      * @return InputfieldWrapper
 85:      *
 86:      *  
 87:      * // static version
 88:      * public static function getModuleConfigInputfields(array $data); 
 89:      * 
 90:      * // non-static version
 91:      * public function getModuleConfigInputfields(); 
 92:      *  
 93:      * // non-static version with optional $data array, if you want it for some reason
 94:      * public function getModuleConfigInputfields(array $data); 
 95:      *  
 96:      * // non-static version with optional InputfieldWrapper as a convenience
 97:      * // note that the "return" statement may be omitted when using the $inputfields param.
 98:      * public function getModuleConfigInputfields($inputfields); 
 99:      * 
100:      */
101: 
102:     /*********************************************************************************
103:      * Return an array defining Inputfields (static or non-static)
104:      * 
105:      * You should use either getModuleConfigArray() or getModuleConfigInputfields(),
106:      * do not use both, as ProcessWire will only recognize one or the other. Likewise,
107:      * you should either use the static version of non-static version, not both. 
108:      *
109:      * See notes for getModuleConfigInputfields() above for benefits and drawbacks
110:      * of static vs. non-static versions. The primary difference between this method
111:      * and that method is that this one returns an array. The format of the array should
112:      * be as shown in InputfieldWrapper::importArray (see InputfieldWrapper.php).
113:      * 
114:      * Whether static or non-static, your 'value' attributes in the array need only 
115:      * represent the default values. ProcessWire will populate the actual values 
116:      * to the resulting Inputfields after the method has been called. This is a benefit
117:      * over the getModuleConfigInputfields() methods. 
118:      *
119:      * @return array
120:      *
121:      * public static function getModuleConfigArray(); // static version
122:      * public function getModuleConfigArray(); // non-static version
123:      */
124: 
125:     /**
126:      * Get a module config property
127:      *
128:      * @param string $key
129:      * @return mixed
130:      *
131:      */
132:     public function __get($key);
133: 
134:     /**
135:      * Set a module config property
136:      *
137:      * @param $key
138:      * @param $value
139:      * @return mixed
140:      *
141:      */
142:     public function __set($key, $value);
143: 
144:     /**
145:      * An optional method you may include in your ConfigurableModule to have ProcessWire
146:      * send the configuration data to it rather than populating the properties individually.
147:      *
148:      * @param array $data Array of data in $key => $value format.
149:      *
150:      * public function setConfigData(array $data);
151:      *
152:      */
153: 
154: }
