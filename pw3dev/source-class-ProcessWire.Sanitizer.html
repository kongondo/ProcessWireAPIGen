   3: /**
   4:  * ProcessWire Sanitizer
   5:  *
   6:  * Sanitizer provides shared sanitization functions as commonly used throughout ProcessWire core and modules
   7:  * 
   8:  * #pw-summary Provides methods for sanitizing and validating user input, preparing data for output, and more. 
   9:  * #pw-use-constants
  10:  * #pw-body = 
  11:  * Sanitizer is useful for sanitizing input or any other kind of data that you need to match a particular type or format. 
  12:  * The Sanitizer methods are accessed from the `$sanitizer` API variable and/or `sanitizer()` API variable/function.
  13:  * For example:
  14:  * ~~~~~~
  15:  * $cleanValue = $sanitizer->text($dirtyValue); 
  16:  * ~~~~~~
  17:  * You can replace the `text()` call above with any other sanitizer method. Many sanitizer methods also accept additional
  18:  * arguments—see each individual method for details. 
  19:  * 
  20:  * ### Sanitizer and input
  21:  * 
  22:  * Sanitizer methods are most commonly used with user input. As a result, the methods in this class are also accessible
  23:  * from the `$input->get`, `$input->post` and `$input->cookie` API variables, in the same manner that they are here. 
  24:  * This is a useful shortcut for instances where you don’t need to provide additional arguments to the sanitizer method.
  25:  * Below are a few examples of this usage:
  26:  * ~~~~~
  27:  * // get GET variable 'id' as integer
  28:  * $id = $input->get->int('id');
  29:  * 
  30:  * // get POST variable 'name' as 1-line plain text
  31:  * $name = $input->post->text('name');
  32:  * 
  33:  * // get POST variable 'comments' as multi-line plain text
  34:  * $comments = $input->post->textarea('comments'); 
  35:  * ~~~~~
  36:  * In ProcessWire 3.0.125 and newer you can also perform the same task as the above with one less `->` level like the
  37:  * example below: 
  38:  * ~~~~~
  39:  * $comments = $input->post('comments','textarea'); 
  40:  * ~~~~~
  41:  * This is more convenient in some IDEs because it’ll never be flagged as an unrecognized function call. Though outside
  42:  * of that it makes little difference how you call it, as they both do the same thing. 
  43:  * 
  44:  * See the `$input` API variable for more details on how to call sanitizers directly from $input.
  45:  * 
  46:  * ### Adding your own sanitizers
  47:  *
  48:  * You can easily add your own new sanitizers via ProcessWire hooks. Hooks are commonly added in a /site/ready.php file, 
  49:  * or from a Module, though you may add them wherever you want. The following example adds a sanitizer method called 
  50:  * `zip()` which enforces a 5 digit zip code:
  51:  * ~~~~~
  52:  * $sanitizer->addHook('zip', function(HookEvent $event) {
  53:  *   $sanitizer = $event->object;
  54:  *   $value = $event->arguments(0); // get first argument given to method
  55:  *   $value = $sanitizer->digits($value, 5); // allow only digits, max-length 5
  56:  *   if(strlen($value) < 5) $value = ''; // if fewer than 5 digits, it is not a zip
  57:  *   $event->return = $value;
  58:  * });
  59:  *
  60:  * // now you can use your zip sanitizer
  61:  * $dirtyValue = 'Decatur GA 30030';
  62:  * $cleanValue = $sanitizer->zip($dirtyValue);
  63:  * echo $cleanValue; // outputs: 30030
  64:  * ~~~~~
  65:  *
  66:  * ### Additional options (3.0.125 or newer)
  67:  * 
  68:  * In ProcessWire 3.0.125+ you can also combine sanitizer methods in a single call. These are defined by separating each 
  69:  * sanitizer method with an understore. The example below runs the value through the text sanitizer and then through the 
  70:  * entities sanitizer:
  71:  * ~~~~~
  72:  * $cleanValue = $sanitizer->text_entities($dirtyValue);
  73:  * ~~~~~
  74:  * If you append a number to any sanitizer call that returns a string, it is assumed to be maximum allowed length. For 
  75:  * example, the following would sanitize the value to be text of no more than 20 characters:
  76:  * ~~~~~
  77:  * $cleanValue = $sanitizer->text20($dirtyValue); 
  78:  * ~~~~~
  79:  * The above technique also works for any user-defined sanitizers you’ve added via hooks. We like this strategy for 
  80:  * storage of sanitizer calls that are executed at some later point, like those you might store in a module config. It
  81:  * essentially enables you to define loose data types for sanitization. In addition, if there are other cases where you
  82:  * need multiple sanitizers to clean a particular value, this strategy can do it with a lot less code than you would 
  83:  * with multiple sanitizer calls. 
  84:  * 
  85:  * Most methods in the Sanitizer class focus on sanitization rather than validation, with a few exceptions. You can 
  86:  * convert a sanitizer call to validation call by calling the `validate()` method with the name of the sanitizer and the
  87:  * value. A validation call simply implies that if the value is modified by sanitization then it is considered invalid
  88:  * and thus it’ll return a non-value rather than a sanitized value. See the `Sanitizer::validate()` and 
  89:  * `Sanitizer::valid()` methods for usage details.

 320:      *  - `allowedExtras (array): Specify extra allowed characters (default=`['-', '_', '.']`).
 321:      * @return string Sanitized value in name format
 322:      * @see Sanitizer::pageName()
 323:      * 
 324:      */
 325:     public function name($value, $beautify = false, $maxLength = 128, $replacement = '_', $options = array()) {
 326:     
 327:         if(!empty($options['allowedExtras']) && is_array($options['allowedExtras'])) {
 328:             $allowedExtras = $options['allowedExtras']; 
 329:             $allowedExtrasStr = implode('', $allowedExtras); 
 330:         } else {
 331:             $allowedExtras = array('-', '_', '.');
 332:             $allowedExtrasStr = '-_.';
 333:         }
 334:     
 335:         $value = $this->nameFilter($value, $allowedExtras, $replacement, $beautify, $maxLength);
 336:         
 337:         if($beautify) {
 338:             
 339:             $hasExtras = false;
 340:             foreach($allowedExtras as $c) {
 341:                 $hasExtras = strpos($value, $c) !== false;
 342:                 if($hasExtras) break;
 343:             }
 344:             
 345:             if($hasExtras) {
 346:                 
 347:                 if(empty($options['allowAdjacentExtras'])) {
 348:                     // replace any of '-_.' next to each other with a single $replacement
 349:                     $value = preg_replace('/[' . $allowedExtrasStr . ']{2,}/', $replacement, $value);
 350:                 }
 351: 
 352:                 if(empty($options['allowDoubledReplacement'])) {
 353:                     // replace double'd replacements
 354:                     $r = "$replacement$replacement";
 355:                     if(strpos($value, $r) !== false) $value = preg_replace('/' . $r . '+/', $replacement, $value);
 356:                 }
 357:     
 358:                 // replace double dots
 359:                 if(strpos($value, '..') !== false) $value = preg_replace('/\.\.+/', '.', $value);
 360:             }
 361:             
 362:             if(strlen($value) > $maxLength) $value = substr($value, 0, $maxLength); 
 363:         }
 364:         
 365:         return $value; 
 366:     }

 500:     /**
 501:      * Sanitize as a ProcessWire page name
 502:      *
 503:      * - Page names by default support lowercase ASCII letters, digits, underscore, hyphen and period. 
 504:      * 
 505:      * - Because page names are often generated from a UTF-8 title, UTF-8 to ASCII conversion will take place when `$beautify` is enabled.
 506:      * 
 507:      * - You may optionally omit the `$beautify` and/or `$maxLength` arguments and substitute the `$options` array instead.
 508:      * 
 509:      * - When substituted, the beautify and maxLength options can be specified in $options as well.
 510:      * 
 511:      * - If `$config->pageNameCharset` is "UTF8" then non-ASCII page names will be converted to punycode ("xn-") ASCII page names,
 512:      *   rather than converted, regardless of `$beautify` setting. 
 513:      * 
 514:      * ~~~~~
 515:      * $test = "Hello world!";
 516:      * echo $sanitizer->pageName($test, true); // outputs: hello-world
 517:      * ~~~~~
 518:      * 
 519:      * #pw-group-strings
 520:      * #pw-group-pages
 521:      *
 522:      * @param string $value Value to sanitize as a page name
 523:      * @param bool|int|array $beautify This argument accepts a few different possible values (default=false): 
 524:      *  - `true` (boolean): Make it pretty. Use this when using a pageName for the first time. 
 525:      *  - `$options` (array): You can optionally specify the $options array for this argument instead.
 526:      *  - `Sanitizer::translate` (constant): This will make it translate non-ASCII letters based on *InputfieldPageName* module config settings. 
 527:      *  - `Sanitizer::toAscii` (constant): Convert UTF-8 characters to punycode ASCII.
 528:      *  - `Sanitizer::toUTF8` (constant): Convert punycode ASCII to UTF-8.
 529:      *  - `Sanitizer::okUTF8` (constant): Allow UTF-8 characters to appear in path (implied if $config->pageNameCharset is 'UTF8'). 
 530:      * @param int|array $maxLength Maximum number of characters allowed in the name.
 531:      *  You may also specify the $options array for this argument instead. 
 532:      * @param array $options Array of options to modify default behavior. See Sanitizer::name() method for available options. 
 533:      * @return string
 534:      * @see Sanitizer::name()
 535:      *
 536:      */
 537:     public function pageName($value, $beautify = false, $maxLength = 128, array $options = array()) {
 538:     
 539:         if(!strlen($value)) return '';
 540:         
 541:         $defaults = array(
 542:             'charset' => $this->wire('config')->pageNameCharset
 543:         );
 544:         
 545:         if(is_array($beautify)) {
 546:             $options = array_merge($beautify, $options);
 547:             $beautify = isset($options['beautify']) ? $options['beautify'] : false;
 548:             $maxLength = isset($options['maxLength']) ? $options['maxLength'] : 128;
 549:         } else if(is_array($maxLength)) {
 550:             $options = array_merge($maxLength, $options);
 551:             $maxLength = isset($options['maxLength']) ? $options['maxLength'] : 128;
 552:         } else {
 553:             $options = array_merge($defaults, $options);
 554:         }
 555:         
 556:         if($options['charset'] !== 'UTF8' && is_int($beautify) && $beautify > self::translate) {
 557:             // UTF8 beautify modes aren't available if $config->pageNameCharset is not UTF8
 558:             if(in_array($beautify, array(self::toAscii, self::toUTF8, self::okUTF8))) {
 559:                 // if modes aren't supported, disable 
 560:                 $beautify = false;
 561:             }
 562:         }
 563:         
 564:         if($beautify === self::toAscii) {
 565:             // convert UTF8 to ascii (IDN/punycode)
 566:             $beautify = false;
 567:             if(strlen($value) > $maxLength) $value = substr($value, 0, $maxLength);
 568:             $_value = $value;
 569:             
 570:             if(!ctype_alnum($value)
 571:                 && !ctype_alnum(str_replace(array('-', '_', '.'), '', $value)) 
 572:                 && strpos($value, 'xn-') !== 0) {
 573:                 
 574:                 do {
 575:                     // encode value
 576:                     $value = $this->punyEncodeName($_value);
 577:                     // if result stayed within our allowed character limit, then good, we're done
 578:                     if(strlen($value) <= $maxLength) break;
 579:                     // continue loop until encoded value is equal or less than allowed max length
 580:                     $_value = substr($_value, 0, strlen($_value) - 1);
 581:                 } while(true);
 582:                 
 583:                 // if encode was necessary and successful, return with no further processing
 584:                 if(strpos($value, 'xn-') === 0) {
 585:                     return $value;
 586:                 } else {
 587:                     // can't be encoded, send to regular name sanitizer
 588:                     $value = $_value;
 589:                 }
 590:             }
 591:             
 592:         } else if($beautify === self::toUTF8) {
 593:             // convert ascii IDN/punycode to UTF8
 594:             $beautify = self::okUTF8;
 595:             if(strpos($value, 'xn-') === 0) {
 596:                 // found something to convert
 597:                 $value = $this->punyDecodeName($value);
 598:                 // now it will run through okUTF8
 599:             }
 600:         }
 601:         
 602:         if($beautify === self::okUTF8) {
 603:             return $this->pageNameUTF8($value);
 604:         }
 605:         
 606:         return strtolower($this->name($value, $beautify, $maxLength, '-', $options));
 607:     }

 626:     /**
 627:      * Sanitize and allow for UTF-8 characters in page name
 628:      * 
 629:      * - If `$config->pageNameCharset` is not `UTF8` then this function just passes control to the regular page name sanitizer.
 630:      * - Allowed UTF-8 characters are determined from `$config->pageNameWhitelist`.
 631:      * - This method does not convert to or from UTF-8, it only sanitizes it against the whitelist. 
 632:      * - If given a value that has only ASCII characters, this will pass control to the regular page name sanitizer. 
 633:      * 
 634:      * #pw-group-strings
 635:      * #pw-group-pages
 636:      * 
 637:      * @param string $value Value to sanitize
 638:      * @param int $maxLength Maximum number of characters allowed
 639:      * @return string Sanitized value
 640:      *
 641:      */
 642:     public function pageNameUTF8($value, $maxLength = 128) {
 643:         
 644:         if(!strlen($value)) return '';
 645:         
 646:         // if UTF8 module is not enabled then delegate this call to regular pageName sanitizer
 647:         if($this->wire('config')->pageNameCharset != 'UTF8') return $this->pageName($value, false, $maxLength);
 648:     
 649:         // we don't allow UTF8 page names to be prefixed with "xn-"
 650:         if(strpos($value, 'xn-') === 0) $value = substr($value, 3);
 651: 
 652:         // word separators that we always allow 
 653:         $separators = array('.', '-', '_'); 
 654:     
 655:         // we let regular pageName handle chars like these, if they appear without other UTF-8
 656:         $extras = array('.', '-', '_', ' ', ',', ';', ':', '(', ')', '!', '?', '&', '%', '$', '#', '@');
 657: 
 658:         // proceed only if value has some non-ascii characters
 659:         if(ctype_alnum(str_replace($extras, '', $value))) return $this->pageName($value, false, $maxLength);
 660: 
 661:         // validate that all characters are in our whitelist
 662:         $whitelist = $this->wire('config')->pageNameWhitelist;
 663:         if(!strlen($whitelist)) $whitelist = false;
 664:         $blacklist = '/\\%"\'<>?#@:;,+=*^$()[]{}|&';
 665:         $replacements = array();
 666: 
 667:         for($n = 0; $n < mb_strlen($value); $n++) {
 668:             $c = mb_substr($value, $n, 1);
 669:             if(!strlen(trim($c)) || ctype_cntrl($c)) {
 670:                 // character does not resolve to something visible
 671:                 $replacements[] = $c;
 672:             } else if(mb_strpos($blacklist, $c) !== false || strpos($blacklist, $c) !== false) {
 673:                 // character that is in blacklist
 674:                 $replacements[] = $c;
 675:             } else if($whitelist !== false && mb_strpos($whitelist, $c) === false) {
 676:                 // character that is not in whitelist, double check case variants
 677:                 $cLower = mb_strtolower($c);
 678:                 $cUpper = mb_strtoupper($c);
 679:                 if($cLower !== $c && mb_strpos($whitelist, $cLower) !== false) {
 680:                     // allow character and convert to lowercase variant
 681:                     $value = mb_substr($value, 0, $n) . $cLower . mb_substr($value, $n+1);
 682:                 } else if($cUpper !== $c && mb_strpos($whitelist, $cUpper) !== false) {
 683:                     // allow character and convert to uppercase varient
 684:                     $value = mb_substr($value, 0, $n) . $cUpper . mb_substr($value, $n+1);
 685:                 } else {
 686:                     // queue character to be replaced
 687:                     $replacements[] = $c;
 688:                 }
 689:             }
 690:         }
 691: 
 692:         // replace disallowed characters with "-"
 693:         if(count($replacements)) $value = str_replace($replacements, '-', $value);
 694:         
 695:         // replace doubled word separators
 696:         foreach($separators as $c) {
 697:             while(strpos($value, "$c$c") !== false) {
 698:                 $value = str_replace("$c$c", $c, $value);
 699:             }
 700:         }
 701: 
 702:         // trim off any remaining separators/extras
 703:         $value = trim($value, '-_.');
 704:         
 705:         if(mb_strlen($value) > $maxLength) $value = mb_substr($value, 0, $maxLength); 
 706:         
 707:         return $value;
 708:     }

 712:      * 
 713:      * @param string $value
 714:      * @return string
 715:      * 
 716:      */
 717:     protected function punyDecodeName($value) {
 718:         // exclude values that we know can't be converted
 719:         if(strlen($value) < 4 || strpos($value, 'xn-') !== 0) return $value;
 720:         
 721:         if(strpos($value, '__')) {
 722:             $_value = $value;
 723:             $parts = explode('__', $_value);
 724:             foreach($parts as $n => $part) {
 725:                 $parts[$n] = $this->punyDecodeName($part);
 726:             }
 727:             $value = implode('', $parts);
 728:             return $value; 
 729:         }
 730:         
 731:         $_value = $value; 
 732:         // convert "xn-" single hyphen to recognized punycode "xn--" double hyphen
 733:         if(strpos($value, 'xn--') !== 0) $value = 'xn--' . substr($value, 3);
 734:         if(function_exists('idn_to_utf8')) {
 735:             // use native php function if available
 736:             $value = @idn_to_utf8($value);
 737:         } else {
 738:             // otherwise use Punycode class
 739:             $pc = new Punycode();
 740:             $value = $pc->decode($value);
 741:         }
 742:         // if utf8 conversion failed, restore original value
 743:         if($value === false || !strlen($value)) $value = $_value;
 744:         return $value;
 745:     }

 748:      * Encode a name value to PW-punycode
 749:      * 
 750:      * @param string $value
 751:      * @return string
 752:      * 
 753:      */
 754:     protected function punyEncodeName($value) {
 755:         // exclude values that don't need to be converted
 756:         if(strpos($value, 'xn-') === 0) return $value;
 757:         if(ctype_alnum(str_replace(array('.', '-', '_'), '', $value))) return $value;
 758: 
 759:         while(strpos($value, '__') !== false) {
 760:             $value = str_replace('__', '_', $value);
 761:         }
 762:         
 763:         if(strlen($value) >= 50) {
 764:             $_value = $value;
 765:             $parts = array();
 766:             while(strlen($_value)) {
 767:                 $part = mb_substr($_value, 0, 12);
 768:                 $_value = mb_substr($_value, 12);
 769:                 $parts[] = $this->punyEncodeName($part);
 770:             }
 771:             $value = implode('__', $parts);
 772:             return $value; 
 773:         }
 774:         
 775:         $_value = $value;
 776:         
 777:         if(function_exists("idn_to_ascii")) {
 778:             // use native php function if available
 779:             $value = substr(@idn_to_ascii($value), 3);
 780:         } else {
 781:             // otherwise use Punycode class
 782:             $pc = new Punycode();
 783:             $value = substr($pc->encode($value), 3);

 788:             $value = "xn-$value";
 789:         } else {

 791:             // return value is always ascii
 792:             $value = $this->name($_value);
 793:         }
 794:         return $value;
 795:     }

1656:     /**
1657:      * Implementation of PHP's FILTER_VALIDATE_URL with IDN and underscore support (will convert to valid)
1658:      *
1659:      * Example: http://трикотаж-леко.рф
1660:      *
1661:      * @param string $url

1663:      * @return string

1669:         $placeholders = array();

1673:             do {
